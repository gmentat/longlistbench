This revision reports corpus validation and four OCR-conditioned agentic extraction baselines on the released corpus of 36 documents. Earlier prerelease experiments used a development set of 80 documents with different templates, schemas, and scoring. We retain those experiments as development history but do not mix their scores with the current release.

\subsection{Agentic OCR baselines}
\label{subsec:current-ocr-baseline}
Each run received one OCR transcript and the public field contract in a temporary workspace. A macOS sandbox denied the benchmark repository, and ground truth, target values and counts, and generator code were absent. Agents could inspect the transcript, write temporary code, validate JSON, and return the complete list. These are protocol baselines, not raw one-shot model measurements.

The initial GPT-5.5 and Opus 4.8 runs were produced on July 1 and July 12, 2026; GPT-5.6-Sol and Fable 5 were run on July 13. After seven document layouts were corrected, those inputs were rerun for all four configurations under the same xhigh protocol, while the 29 unchanged-document predictions were retained. The current runners used Codex CLI 0.144.1 and Claude Code 2.1.207. Claude Code safe mode disabled project instructions, skills, plugins, hooks, and MCP servers. Its usage metadata listed Fable 5 for every Fable run and auxiliary Haiku 4.5 routing, so this is a Claude Code/Fable 5 protocol rather than a raw-model score. The failures differ in kind. Both agents return all 1{,}260 driver rows, but only 24 are exact because sparse MVR fields are not joined back into the roster; field micro-$F_1$ is still 89.1\%. On split return schedules, GPT-5.6-Sol returns 4{,}326 of 4{,}923 rows and Fable 5 returns 4{,}779, with additional field errors. On page 18 of \path{ifta_tax_inquiry_002}, Gemini converted a card grid to CSV and omitted 23 visible row-label prefixes; this transcript loss explains most of the shared 98.1\% family score and is not solely an agent error. Both agents now complete the BOP and WC policy lists after the policy schedules were corrected, but the CGL packet remains incomplete at 98.5\% and 92.4\% exact recall. In contrast, both recover all 77 long-range claim records exactly, and each misses only one of 796 cross-section return records.

Cross-cutting slices reinforce this variation. Exact-record recall for GPT-5.6-Sol and Fable 5 is 99.4\% and 96.8\% on heterogeneous lists, 99.4\% and 97.0\% on long-range evidence, 84.7\% and 86.7\% on split records, 84.8\% and 86.7\% on inherited context, and 81.7\% and 84.4\% under layout randomization. Duplicate-bearing documents score 99.1\% and 97.8\%. Cross-section joins and repeated keys score 99.9\% for both. These are observational associations because tags overlap and document families differ; paired renderings would be needed to estimate one stressor's causal effect.

Full-context one-shot extraction is not a practical full-corpus protocol here. Small claim packets can return scoreable outputs, but larger outputs reach model limits or time out. We therefore treat one-shot prompting as a lower-bound operational stress test and use the per-document agentic protocol for full-corpus comparison.
