This revision reports corpus validation and four OCR-conditioned agentic extraction baselines on the released corpus of 36 documents. Earlier prerelease experiments used a development set of 80 documents with different templates, schemas, and scoring. We retain those experiments as development history but do not mix their scores with the current release.

\subsection{Agentic OCR baselines}
\label{subsec:current-ocr-baseline}
Each run received one OCR transcript and the public field contract in a temporary workspace. A macOS sandbox denied the benchmark repository, and ground truth, target values and counts, and generator code were absent. Agents could inspect the transcript, write temporary code, validate JSON, and return the complete list. These are protocol baselines, not raw one-shot model measurements.

The two earlier runs used Codex CLI with \texttt{gpt-5.5} at xhigh reasoning on July 1, 2026, and Claude Code with \texttt{claude-opus-4-8} at xhigh effort on July 12. The new runs used Codex CLI 0.144.1 with \texttt{gpt-5.6-sol} and Claude Code 2.1.207 with \texttt{claude-fable-5}, both at xhigh effort on July 13. Claude Code safe mode disabled project instructions, skills, plugins, hooks, and MCP servers. Its usage metadata listed Fable 5 for every run and auxiliary Haiku 4.5 routing, so this is a Claude Code/Fable 5 protocol rather than a raw-model score. All four runs cover the same manifest and 33{,}450 targets with zero execution errors; saved predictions reproduce every score in \Cref{tab:agentic-full-corpus}.

The failures differ in kind. Both agents return all 1{,}260 driver rows, but only 24 are exact because sparse MVR fields are not joined back into the roster; field micro-$F_1$ is still 89.1\%. On split return schedules, GPT-5.6-Sol returns 4{,}326 of 4{,}923 rows and Fable 5 returns 4{,}779, with additional field errors. Policy packets expose model-specific behavior: GPT-5.6-Sol omits 192 BOP records, while Fable returns the full policy record counts but still misses fields. In contrast, both agents recover all 77 long-range claim records exactly, and each misses only one of 796 cross-section return records.

Cross-cutting slices reinforce this variation. Exact-record recall for GPT-5.6-Sol and Fable 5 is 78.7\% and 92.5\% on heterogeneous lists, 79.8\% and 92.9\% on long-range evidence, 82.6\% and 86.4\% on split records, 82.7\% and 86.4\% on inherited context, and 81.9\% and 84.6\% under layout randomization. Duplicate-bearing documents score 86.6\% and 95.2\%. By contrast, cross-section joins and repeated keys score 99.9\% for both. These are observational associations because tags overlap and document families differ; paired renderings would be needed to estimate one stressor's causal effect.

Full-context one-shot extraction is not a practical full-corpus protocol here. Small claim packets can return scoreable outputs, but larger outputs reach model limits or time out. We therefore treat one-shot prompting as a lower-bound operational stress test and use the per-document agentic protocol for full-corpus comparison.
