# Benchmarks

This directory contains the OCR, indexing, evaluation, and Hugging Face export tools for LongListBench. The public dataset artifacts live under `data/`; the current release is a generated corpus, not a recipe that should be regenerated by default.

## Current Dataset

The current `data/` release contains:

| Split/config | PDFs | Target records | Notes |
|---|---:|---:|---|
| `core_operations` | 28 | 31,088 | Production-like IFTA, driver/MVR, vehicle schedule, and loss-run layouts. |
| `claim_multihop` | 3 | 77 | Long claim PDFs requiring cross-page joins. |
| `policy_packets` | 5 | 1,945 | Two dense BOP declaration schedules plus long BOP, WC, and CGL policy packets. |
| **Total** | **36** | **33,110** | Every PDF has OCR and JSON ground truth. |

The released transcript condition is `ocr`. The current corpus does not ship clean structural transcripts.

`data/manifest.json` is the source of truth for sample IDs, artifact paths, target counts, transcript availability, and provenance notes.

## Setup

Run these commands from the repository root.

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r benchmarks/requirements.txt
python -m playwright install chromium
```

PDF/OCR tooling also needs Poppler:

```bash
# macOS
brew install poppler

# Linux
apt-get install poppler-utils
```

API keys are only needed for OCR or model evaluation:

```bash
cp .env.example .env
```

Common variables:

```bash
VERTEX_AI_API_KEY=your-gemini-or-vertex-api-key
GEMINI_API_KEY=your-gemini-api-key
OPENAI_API_KEY=your-openai-api-key

# Optional evaluation controls
LLB_MODEL_WORKERS=2
LLB_GEMINI_CHUNK_MAX_INPUT_TOKENS=12000
```

Install Hugging Face export dependencies only when packaging for the Hub:

```bash
python -m pip install -r benchmarks/requirements-hf.txt
```

```text
  ground_truth/<sample_id>.json
  transcripts/ocr_gemini/<sample_id>.md
  metadata/<sample_id>.json
  schemas/*.schema.json
```

Regenerate the browsable index after changing artifacts or metadata:

```bash
python benchmarks/build_instance_index.py --input data
```

## OCR

Process all PDFs with Gemini OCR:

```bash
python benchmarks/ocr_claims_pdfs.py --model gemini-3.5-flash --force
```

The script writes transcripts under `data/transcripts/ocr_gemini/{sample_id}.md`, updates manifest transcript availability, and skips existing OCR files unless `--force` is provided. After any PDF replacement, rerun OCR; transcripts from older PDFs are not reusable.

If using OpenRouter for Gemini OCR:

```bash
python benchmarks/ocr_claims_pdfs.py \
  --ocr-engine openrouter \
  --model google/gemini-3.5-flash \
  --force
```

## Evaluation

Run extraction evaluation across registered model/regime keys. Common keys include `gemini`, `gemini_oneshot`, `gpt52`, `gpt55_oneshot`, `gpt55_chunked`, and `gpt55_agent`.

```bash
# OCR-condition evaluation
python benchmarks/evaluate_models.py --models gpt55_oneshot --transcripts ocr

# GPT-5.5 agentic extraction on the multi-hop OCR cases
LLB_AGENT_REASONING_EFFORT=xhigh LLB_AGENT_VERBOSITY=high \
python benchmarks/evaluate_models.py \
  --models gpt55_agent \
  --tiers multihop mixed \
  --transcripts ocr \
  --output-dir benchmarks/results/agentic_multihop_gpt55

# Regenerate a report offline from an existing results directory
python benchmarks/evaluate_models.py --offline --output-dir benchmarks/results/scratch --transcripts ocr
```

Results are written to the `--output-dir`:

- `evaluation_report.json` - full metrics data.
- `evaluation_report.md` - human-readable summary.

Saved reports under `benchmarks/results/` may refer to earlier corpus versions. Do not cite them as current-layout baselines unless they have been rerun against the current `data/` manifest.

## Hugging Face Export

Package the current `data/` directory as a Hugging Face dataset repository:

```bash
python -m pip install -r benchmarks/requirements-hf.txt
python benchmarks/export_hf_dataset.py \
  --input data \
  --output dist/huggingface/longlistbench \
  --repo-id kaydotai/LongListBench \
  --overwrite
```

The export writes:

- `README.md` - Hugging Face dataset card.
- `data/core_operations/test-00000-of-00001.parquet` - 28 core operations PDFs.
- `data/claim_multihop/test-00000-of-00001.parquet` - 3 cross-page claim PDFs.
- `data/policy_packets/test-00000-of-00001.parquet` - 5 policy PDFs.
- `schemas/*.schema.json` - public extraction schemas.
- `metadata/manifest.json` - source manifest copied from `data/`.

Each Parquet row includes `document_id`, `domain`, `complexity_regime`, `document_format`, `target_field`, `target_record_type`, `target_count`, embedded `pdf`, JSON-string `ground_truth`, JSON-string `metadata`, and `ocr_transcript`. Claim multi-hop rows use `target_field="incidents"`; operations, external loss-run, and policy rows use `target_field="records"`.

To upload after inspection:

```bash
python benchmarks/export_hf_dataset.py \
  --input data \
  --output dist/huggingface/longlistbench \
  --repo-id kaydotai/LongListBench \
  --overwrite \
  --upload
```

## Provenance

The repository contains synthetic generated artifacts only. Private production PDFs were used as visual layout references during template design, but production documents, names, account numbers, and raw customer data are not included in the repo.

The generator scripts that produced the current production-like templates are intentionally kept outside the repository. The repo ships the dataset artifacts and evaluation harness.
